# 1.4 单文件架构 + 移动端适配

## 问题来源

两个独立但相关的问题，在同一次迭代中一并解决：

**问题一：多页网站无法在 Claude Code 右侧预览**。之前多页站会生成多个 `.html` 文件（`index.html`、`about.html`、`contact.html`…），Claude Code 的 preview 工具只能挂载单一入口，导致多页网站无法利用右侧预览面板验证效果，只能让用户自己去本地双击文件打开。

**问题二：生成的网站没有移动端适配，还用折叠导航**。Step 7 的生成约束里从未要求写 `@media` 移动端规则，Step 8 的交付检查也没有移动端验证环节。更严重的是，之前 technical-spec.md 的导航栏模板默认使用汉堡菜单（hamburger icon + 抽屉）——这个方案在手机屏幕上的实际使用体验极差，导航项藏起来用户根本找不到。

---

## 本次迭代核心思路

**单文件架构**：无论单页还是多页网站，都只生成一个 `index.html`，CSS/JS 全部内联，多页通过 hash 路由（`#home`、`#about`）切换。这样做一次解决两个问题：Claude Code preview 工具能直接挂载；用户拿到的产物也更简单，双击一个文件就够了。

**移动端必须平铺导航**：导航是用户在手机上操作网站的核心入口，折叠菜单等于把入口藏起来。改为强制平铺方案——导航项在移动端换行或水平滚动，始终可见，无需任何点击才能展开。

## 迭代内容

### 1. 单文件架构（SKILL.md Step 7 + technical-spec.md）

#### 1.1 核心规则

所有项目只生成 `index.html` 一个 HTML 文件，不生成独立的 `.css` 或 `.js` 文件：

- **CSS 内联在 `<style>` 标签中**，用 CSS class 组织样式（`.hero`、`.card`、`.features`），禁止大段重复的 `style=""` 属性
- **JS 内联在 `<script>` 标签中**
- **多页网站用 hash 路由**：每个「页面」是 `<div class="page" id="page-xxx">`，通过 JS 监听 `hashchange` 显示/隐藏；导航链接用 `href="#pagename"` 格式

#### 1.2 Hash 路由标准实现

```html
<!-- 导航 -->
<a href="#home">首页</a>
<a href="#about">关于</a>

<!-- 页面容器 -->
<div class="page active" id="page-home">...</div>
<div class="page" id="page-about">...</div>

<!-- CSS -->
<style>
.page { display: none; }
.page.active { display: block; }
</style>

<!-- JS -->
<script>
function router() {
  const hash = location.hash.slice(1) || 'home';
  document.querySelectorAll('.page').forEach(p => p.classList.remove('active'));
  const target = document.getElementById('page-' + hash);
  if (target) {
    target.classList.add('active');
    window.scrollTo(0, 0);
  }
  document.querySelectorAll('.nav__links a[href^="#"]').forEach(a => {
    a.classList.toggle('active', a.getAttribute('href') === '#' + hash);
  });
}
window.addEventListener('hashchange', router);
window.addEventListener('DOMContentLoaded', router);
</script>
```

导航栏和页脚**只写一次**，放在所有 `.page` 容器的外面（全局共享）。

```
之前（多页）：            现在（单文件）：
site/                    site/
├── index.html           ├── index.html   ← 包含所有页面

```

### 2. 移动端适配（SKILL.md Step 7 + technical-spec.md）

#### 2.1 导航栏：禁止折叠/汉堡菜单

**移动端导航必须平铺显示所有导航项**，不使用 hamburger icon 或任何折叠方案。

两种平铺方案（根据导航项数量选择）：

**换行平铺**（导航项 ≤ 5 个）：
```css
@media (max-width: 768px) {
  .nav__inner { flex-wrap: wrap; justify-content: center; gap: 0.5rem; padding: 0.75rem 1rem; }
  .nav__brand { width: 100%; text-align: center; }
  .nav__links { flex-wrap: wrap; justify-content: center; gap: 0.5rem 1rem; }
  .nav__links a { font-size: 0.85rem; padding: 0.35rem 0.6rem; }
}
```

**水平滚动**（导航项 > 5 个）：
```css
@media (max-width: 768px) {
  .nav__links {
    flex-wrap: nowrap;
    overflow-x: auto;
    -webkit-overflow-scrolling: touch;
    scrollbar-width: none;
    padding-bottom: 0.25rem;
  }
  .nav__links::-webkit-scrollbar { display: none; }
  .nav__links li { flex-shrink: 0; }
}
```

#### 2.2 布局适配规则（新增至 technical-spec.md）

```css
@media (max-width: 768px) {
  /* 多列降为单列 */
  .grid-2col, .grid-3col, .grid-4col { grid-template-columns: 1fr; }
  /* 图文并排改上下堆叠 */
  .split-layout { flex-direction: column; }
  /* 标题响应式字号 */
  h1 { font-size: clamp(1.75rem, 6vw, 3rem); }
  h2 { font-size: clamp(1.35rem, 4vw, 2.25rem); }
  /* Section 内边距缩小 */
  section { padding: 3rem 1rem; }
  /* Hero 适配 */
  .hero { min-height: 80vh; padding: 4rem 1rem 3rem; }
  /* 按钮全宽 */
  .btn { width: 100%; text-align: center; }
}
```

#### 2.3 触控友好规范

- 所有可点击元素最小尺寸 **44×44px**（WCAG 2.5.5）
- 按钮和链接之间最小间距 **8px**，防止误触
- 输入框高度 ≥ **48px**

### 3. Step 7 生成约束更新

在 9 条约束之后新增**第 10 条**移动端适配必须项：

```
10. 移动端适配（必须在每个页面区块中实现）：
    - 导航栏禁止折叠/汉堡菜单：移动端平铺显示所有导航项
    - 多列网格（2col/3col/4col）在 ≤768px 时降为单列
    - 图文并排布局在移动端改为上下堆叠
    - 标题使用 clamp() 实现响应式字号
    - 按钮在移动端全宽显示
    - 所有可点击元素最小 44×44px
    - Hero 区 min-height: 80vh，内边距缩减
```

---

### 4. Step 8 交付检查新增移动端验证

原 5 项检查扩展为 6 项，新增**第 ⑤ 步**移动端适配检查（原 ⑤ 顺延为 ⑥）：

```
⑤ 移动端适配检查
用 preview_resize 切换到 375×812，逐项验证：
1. 导航是否平铺显示（不能出现汉堡菜单/折叠图标）
2. 多列网格是否已降为单列
3. 图文并排是否已改为上下堆叠
4. 标题字号是否正常缩放（不溢出、不过小）
5. 按钮是否全宽可点、间距合理
6. Hero 区高度和内边距是否适配
7. preview_screenshot 截取移动端首屏截图
8. 验证完毕后 preview_resize 切回桌面端尺寸（1280×800）
```
