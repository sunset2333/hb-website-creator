# 1.6 修复 portal wrapper 导致首屏空白

## 问题描述

生成的 `widget.jsx` 在主 `Widget` 组件返回值中，用 `createCustomMountPointPortal` 渲染 Navbar 时，
外层多包了一层 `<div className="site-widget">`：

```jsx
// ❌ 错误写法（1.5 及之前版本）
{createCustomMountPointPortal(
  <div className="site-widget">
    <Navbar page={page} navigate={navigate} props={props} />
  </div>
)}
```

**根因**：mock 环境下 `createCustomMountPointPortal` 直接内联返回组件（不是真正的 portal），
这个内层 `<div className="site-widget">` 有 `min-height: 100vh`，但其唯一子元素 `<nav>` 是
`position: fixed` 不参与文档流，导致撑出一整屏空白，把 Hero 区域完全推到视口外。
用户看到的现象：首屏全黑/空白，内容在页面底部才出现。

## 修复内容

### 1. `references/technical-spec.md`

去掉 portal 调用处多余的 `.site-widget` wrapper，并添加警告注释说明原因：

```jsx
// ✅ 正确写法（1.6+）
{createCustomMountPointPortal(
  <Navbar page={page} navigate={navigate} props={props} />
)}
```

### 2. `assets/template/mock/page-client-toolkit.jsx`

将 `createCustomMountPointPortal` 从「直接返回组件」改为「用 `ReactDOM.createPortal` 渲染到
body 下独立容器」，真正模拟生产环境的 portal 行为：

```jsx
// ✅ 修复后的 mock
let _portalContainer = null;
function getPortalContainer() {
  if (!_portalContainer) {
    _portalContainer = document.createElement('div');
    _portalContainer.id = 'hb-mount-portal';
    document.body.appendChild(_portalContainer);
  }
  return _portalContainer;
}

export function createCustomMountPointPortal(component) {
  if (typeof document === 'undefined') return null;
  return ReactDOM.createPortal(component, getPortalContainer());
}
```

## 影响范围

- **新建项目**：自动生效（模板已更新）
- **已有项目**：将 `mock/page-client-toolkit.jsx` 替换为新版，并将 `widget.jsx` 中
  `createCustomMountPointPortal` 调用处的 `<div className="site-widget">` wrapper 去掉即可
