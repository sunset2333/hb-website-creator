# 1.4 增加图片素材策略+去 AI 味提示词

## 问题来源

通过多次实际生成观察到的核心问题：

**生成的网站全是文字，没有图片，不精致。** 无论快速模式还是标准模式，产出的页面 Hero 区是纯色背景+标题，产品区是卡片+文案，关于区是文字段落——整站像一份排版稍好的 Word 文档，而不是一个有视觉冲击力的官网。

根本原因有三个：

1. **流程中没有图片素材环节**：从 Step 0 到 Step 5，没有任何步骤问用户要图片，也没有兜底方案
2. **生成约束不要求使用图片**：Step 5 的 7 条约束里，没有一条要求 Hero 必须有背景图、section 必须有配图
3. **视觉设计缺乏反AI感规则**：配色保守（浅灰底+蓝按钮）、布局对称均匀、动效千篇一律的 fade-in，缺少大胆的设计决策

附带问题：Codex 等外部工具生成的质量更差（布局混乱、色彩不协调、完全没有设计感），说明 skill 本身需要更强的设计约束来拉开差距。

---

## 本次迭代核心思路

**Unsplash 优先占位 + 给出换图标准，让用户后续自己替换真实素材。**

传统思路是"先收集用户素材→再开始设计"，但实际场景中大部分用户手上没有现成的高质量图片，等用户准备素材会大幅拉长交付周期。

新策略：
- **按产品类型分流视觉策略**：实体产品/线下服务用 Unsplash 摄影占位；软件/SaaS/App 用内联 SVG 插画 + CSS 设备框 + 渐变背景，不用实拍照片填满页面
- 优先用占位素材快速出可视化效果，同时给出每张图的**换图标准**（尺寸、构图、情绪），用户后续按标准替换
- 如果用户主动提供了图片，评估是否符合使用标准，不符合的给出改进建议并替代

同时从 Anthropic 官方 frontend-design skill 和社区"去AI味"实践中提炼反AI感设计规则，融入 Step 4 和 Step 5。

## 迭代内容

### 1. 图片素材交互策略（修改 Step 0/1 各模式入口）

#### 1.1 交互设计：Unsplash 优先

在各模式的问答环节中，图片相关引导改为：

```
**图片素材**：
我会先用 Unsplash 免费图库帮你找合适的配图，让你先看到整体效果。
每张图我会标注替换标准（尺寸、构图要求），你后续可以用自己的真实照片替换。

→ A. 先用占位图，我后续再换（推荐）
→ B. 我现在就有图片素材，放在 [文件夹路径]
→ C. 我有一些图但不确定能不能用，你帮我看看
```

**选 A**（推荐路径）：Claude 在 Step 5 生成时，用 WebSearch 搜 Unsplash + curl 下载到 assets/，交付时附上每张图的换图标准。

**选 B**：Claude 用 `ls` 列出文件夹内容，根据文件名/用户描述分配图片用途，不逐张 Read（省 token）。

**选 C**：Claude Read 用户提供的图片，评估是否满足使用标准（分辨率 ≥ 1200px 宽、构图合适、与品牌调性一致），不满足的给出具体建议并用 Unsplash 替代。

#### 1.1.5 按产品类型分流视觉策略

图片策略不再一刀切，根据产品类型选择不同方案：

| 产品类型 | 视觉策略 | 典型场景 |
|---------|---------|---------|
| 实体产品/线下服务 | 摄影驱动型 → Unsplash 占位 | 餐饮、健身、教育、房产、制造业 |
| 软件/SaaS/App | 产品展示型 → SVG插画 + CSS设备框 + 渐变 | 管理系统、效率工具、开发者平台 |
| 混合型 | 摄影+产品结合 | 有线下场景的科技公司 |

**软件/SaaS 产品的视觉元素优先级**：
1. **内联 SVG 插画**（最推荐）：Claude 直接在 JSX 中绘制，颜色用 CSS 变量自动跟随品牌配色，零外部依赖
2. **CSS 设备框 + 产品截图占位**：纯 CSS 浏览器窗口/手机框，用户后续替换真实截图
3. **CSS/SVG 渐变与几何背景**：gradient mesh、几何图案，代替全幅照片
4. **unDraw 开源插画**（undraw.co）：免费商用无需署名，可自定义颜色匹配品牌

**软件产品的布局约束**：
- 禁止全部 section 用全幅大图 —— 至少 50% 使用「左文右图」或「图文交替」布局
- Hero 区推荐左侧文案 + 右侧产品截图（CSS 设备框），不等分布局
- 功能展示区推荐奇数行左文右图、偶数行左图右文的交替排列

#### 1.2 各模式插入位置

| 模式 | 插入位置 | 方式 |
|------|---------|------|
| 快速 A | Step 0 快速版的 5 问之后，增加第 6 问 | 合并到问答中 |
| 标准 B | Step 0+0.5 合并版的 Q2 素材问题中 | 扩展现有素材问题 |
| 深度 C | Step 1B 素材解读的素材清单中 | 扩展现有图片项 |

#### 1.3 人物照片约束

**除非内容明确面向海外/国际受众，人物照片必须使用亚洲面孔。**

- Unsplash 搜索时加 "asian" 关键词（如 "asian business team meeting"）
- 禁止出现典型 stock photo 感：白背景摆拍、职业微笑、西装握手
- 搜索优先级：真实工作场景 > 生活方式 > 肖像特写

#### 1.4 Unsplash 搜索执行方式

不需要 API key，不需要写脚本。Claude 在 Step 5 生成阶段内联操作：

```
1. WebSearch "site:unsplash.com {行业关键词} {场景关键词} {情绪关键词}"
   → 英文关键词效果更好，根据品牌定位组合
2. WebFetch Unsplash 页面 → 提取图片原始 URL
3. curl -L -o assets/{page}_{element}_{desc}.jpg "{unsplash_url}?w=1600&fit=crop"
   → 下载到 assets/ 目录
```

每张图约 3 次工具调用、~850 tokens。一个网站 5-8 张图总计 ~5000-7000 tokens。

Step 6 交付时，每张 Unsplash 占位图附上换图标准卡：

```
## 图片替换指南

| 位置 | 当前文件 | 替换标准 |
|------|---------|---------|

| 产品展示 | product_frame_1.jpg | 宽 ≥ 1200px，产品特写，干净背景，展示细节 |
| ... | ... | ... |

替换方法：将新图片重命名为对应文件名，放入 assets/ 目录覆盖即可。
```

### 3. 反AI感设计规则（修改 Step 4 + Step 5）

#### 3.1 Step 4 新增设计约束

在 4.3 七支柱之后增加反AI感设计规则：

**字体**：
- 禁止 Inter/Roboto/Arial/system-ui 作为主字体（technical-spec.md 已有字体表，严格按风格匹配选择）
- 标题字体与正文字体必须形成对比（粗细、字体家族、大小至少差 2 级）
- 连续两个项目不得使用相同字体组合

**色彩**：
- 禁止"安全配色"：浅灰底+蓝色按钮、白底+紫色渐变、全站只有黑白灰
- 主色必须大胆占主导地位，辅色用于锐利点缀（accent），不均匀分配
- 设计文档创意声明必须写明"为什么不用这个行业最常见的配色"
- 背景 ≠ 白色或 #f5f5f5——可以用深色、带色调的浅色、渐变、微纹理

**空间构成**：
- 至少 1 个 section 使用非对称布局（文字偏左/右 + 图片不等分、对角线流动）
- 至少 1 个元素打破网格（跨列、图片出血到边缘、元素重叠）
- section 间距不得全部相同——至少 3 种不同的垂直节奏

**背景与氛围**：
- Hero 区禁止纯色背景——必须有以下至少一种：全幅背景图 + overlay、渐变网格、噪点纹理、几何图案
- 全站至少 2 个 section 使用非纯色背景
- 可选装饰：grain overlay、SVG 几何装饰、品牌纹样

**动效**：
- 首屏入场必须 staggered reveal：标题 → 0.2s 副标题 → 0.4s CTA → 0.6s 装饰元素（不是所有元素同时 fade-in）
- 按钮 hover 必须有 transform + box-shadow 组合变化
- 卡片 hover 必须有 translateY(-4px) + shadow 加深

**反趋同声明**（写入设计文档创意声明）：
- 写出"这个设计不像典型AI生成网站的 3 个理由"
- 每个理由对应一个具体的、可在代码中验证的设计决策

#### 3.2 Step 5 新增生成约束

在现有 7 条约束之后增加：

```
8. **图片使用**：
   - Hero section 必须使用全幅背景图（background-image + 半透明遮罩层确保文字可读）
   - 每页至少 1 个 section 包含图片元素（<img> 或 background-image）
   - 如设计文档有照片分隔带，使用 background-attachment: fixed 视差效果
   - 所有图片通过 props.media 引用，在 main.jsx 中声明映射

9. **反AI感实施**：
   - 首屏入场动画使用 staggered animation-delay（具体延迟值见设计文档）
   - 至少 1 个 section 使用 CSS Grid 不等分列（如 grid-template-columns: 1fr 1.5fr）
   - 背景处理：Hero 用 linear-gradient overlay + background-image；至少 1 个中间 section 用渐变或纹理
   - section 之间 padding 至少有 3 种不同值（如 6rem / 8rem / 10rem），禁止全部相同
   - 按钮/卡片 hover 必须有 transform + box-shadow 组合效果
```

### 5. Step 6 交付检查新增图片项

在现有检查 ⑤ 之后增加：

```
**⑤.5 图片完整性检查**
- 确认 assets/ 目录包含设计文档图片策略中规划的所有图片文件
- 确认 Hero section 使用了背景图（不是纯色背景）
- 确认每个页面至少有 1 个 section 包含图片元素
- 确认 Unsplash 图片已下载到 assets/ 并正确命名
- 确认 main.jsx 的 media 映射覆盖所有 assets/ 中的图片
```

```

✅ 链接配置完整
✅ 图片完整（X 张图片，Hero 背景图已配置，Unsplash 占位图附换图标准）

📷 图片替换指南：
[换图标准表格]
```

### 6. 每种 Section 的最低图片要求（新增参考规则）

| Section 类型 | 最低图片数 | 说明 |
|-------------|-----------|------|
| Hero（首屏） | 1 张全幅背景图 | 必须有视觉冲击力，禁止纯色/纯渐变背景 |
| 关于/品牌故事 | 1-2 张 | 主理人照/团队照/工作场景 |
| 产品/服务展示 | 每项 1 张 | 产品实拍或使用场景图 |
| 案例/客户证言 | 每个案例 1 张 | 客户照片、项目截图或成果图 |
| 照片分隔带 | 1 张全幅 | 氛围图，与品牌调性一致，可做视差滚动 |
| 数据/成就区 | 可选 | 有背景图则更有层次感 |
| Logo 墙 | 6+ 个 Logo | 灰度统一处理 |
