# static-landing-pages Skill 优化记录

## 背景

对 `/Users/xuhongyuan/Desktop/skill/static-landing-pages` 进行全面审计，覆盖架构、逻辑流程、Token 效率、内容冗余等维度。

该 Skill 总计 ~111KB 文本（SKILL.md 20KB + technical-spec.md 24KB + design-guide.md 36KB + case-library.md 21KB + 模板 ~12KB），是一个六步流程的落地页生成器。

---

## 本次迭代核心思路

**按需加载**——把 references 目录从"每次全量加载"改为"用到什么读什么"。

原来的问题：design-guide.md（36KB）和 case-library.md（21KB）在每次生成时被完整加载进上下文，但单次对话通常只用到其中 1-2 个组合、1-2 个案例、2-3 种交互功能。大量内容是无效上下文，白白消耗 token。

解决方式：将大文件拆分为独立小文件（cases/、combos/、styles/、interactions/），在各自的索引文件里只保留一张摘要表，AI 按需读取具体文件。单次对话可节省 8K-15K tokens 的无关上下文。

## 已完成的迭代

### Token 优化（P0）

**1. 拆分 references 大文件**
- `case-library.md`（21KB）→ 保留为索引，13 个案例拆入 `references/cases/001-vera.md` 等独立文件

- 效果：索引文件各自缩至 2-3KB，AI 按需读取 1-2 个具体文件，而非全量加载

**2. 消除 SKILL.md 与 technical-spec.md 内容重复**
- SKILL.md 只保留流程逻辑和决策点，技术细节指向 technical-spec.md
- 合并 Step 6 交付检查与 technical-spec.md 质量检查清单

**3. 模板代码去重**
- technical-spec.md 中删除约 280 行重复的 widget.jsx / style.less / main.jsx 完整代码
- 改为："从 `/Users/xuhongyuan/Desktop/hb-website-preview/template/` 复制骨架，按以下规则修改"，模板文件作为 single source of truth

**4. 交互功能代码按需加载**
