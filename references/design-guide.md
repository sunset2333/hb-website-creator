# 设计决策指南

## 使用方式

在 Step 4 生成设计文档时，按五步使用：
1. **定位约束矩阵** → 确定设计必须/避免（WHO — 主体与受众）
2. **视觉风格选择** → 确定美学方向（HOW IT LOOKS — 视觉语言）
3. **页面积木库** → 确定页面数量和组合（WHAT — 结构）
4. **Section 布局模式库** → 规划每页内部节奏（RHYTHM — 节奏）
5. **转化与信任设计规则** → 确保转化要素达标（CONVERT — 商业目标）

同时参考 [case-library.md](case-library.md) 执行**反趋同检查**。

---

## ⚡ 两步快速诊断（先用这个，再看矩阵）

**Step 1：主体是人还是公司？**
- 以个人为核心价值（顾问/老师/主理人/创作者）→ 看受众类型
- 以产品/服务体系为核心（企业/品牌/SaaS）→ 看服务性质

**Step 2：核心信任靠什么？**
| 信任来源 | 对应组合 |
|---------|---------|
| 故事/人格感染力 | 组合一（温暖叙事）/ 组合七（社群共创）|
| 数据/规模背书 | 组合二（科技专业）/ 组合四（商务信任）|
| 作品/案例展示 | 组合三（感官体验）/ 组合六（高端精品）|
| 专业权威/资质 | 组合五（知识专家）|
| 安全/保障/孝心 | 组合八（高信任孝心）|

两步定位完成后，直接读取对应组合文件查看设计约束。

## 第一步：定位约束矩阵

### 三个输入维度

**维度一 · 主体类型**
- A1 个人IP（以个人为核心价值）
- A2 品牌公司（以产品/服务体系为核心）
- A3 个人+品牌双轨（个人形象驱动品牌）

**维度二 · 受众类型**
- B1 B端企业客户（理性决策，关注ROI）
- B2 B2C大众（情感驱动，价格敏感）
- B3 B2C高净值个人（品质导向，体验敏感）
- B4 混合（B端+B2C并存）

**维度三 · 信任建立方式**（可多选）
- C1 数据背书（数字、认证、规模）
- C2 故事共鸣（个人经历、价值观叙事）
- C3 作品/案例展示（看得见的成果）
- C4 社群氛围（用户社区、归属感）
- C5 专业权威（资质、流程、体系）

## 第二步：视觉风格选择

定位矩阵解决"必须/避免"的约束，视觉风格解决"具体长什么样"。两者结合，确保设计既有方向约束又有落地美学。

### 10 种核心风格

---

#### S1 · 简约现代（Minimalist Modern）

**核心特征**：大量留白、极有限配色（2-3色）、每个元素有明确功能、无装饰性元素
**最佳搭配**：组合六高端精品、科技产品
**不宜搭配**：信息密度型业务（B端多产品线）、活力品牌

---

#### S2 · 企业专业（Corporate Professional）

**核心特征**：干净结构化布局、无衬线字体、蓝/灰/白为主、严格网格层级、信息组织清晰
**最佳搭配**：组合二科技专业、组合四商务信任、组合五知识专家
**不宜搭配**：个人IP感强、情感驱动型业务

---

#### S3 · 温暖人文（Warm Humanistic）

**核心特征**：低饱和暖色、衬线标题增加温度、大留白长文叙事、photo strip 贯穿、手写感元素可选
**最佳搭配**：组合一温暖叙事、组合三感官体验
**不宜搭配**：科技企业、数据驱动型产品

---

#### S4 · 科技暗黑（Tech Dark）

**核心特征**：深色（#0A-#1A）背景、高对比度点缀色（金/青/霓虹）、精密排版、数据可视化、粒子/光效
**最佳搭配**：组合二科技专业
**不宜搭配**：温暖亲切型、老年受众

---

#### S5 · 摄影沉浸（Photography Immersive）

**核心特征**：全屏英雄图片作主要叙事工具、Ken Burns / 视差动效、文字是图片的注释、大面积高质量照片
**最佳搭配**：旅游/餐饮/房产、组合八孝心型、组合三感官体验
**不宜搭配**：缺少高质量照片的项目、纯文字内容型

---

#### S6 · 编辑杂志（Editorial Magazine）

**核心特征**：大号装饰性标题 + 小号正文、强烈排版对比、大胆摄影、混合字重/字号
**最佳搭配**：内容创作者、媒体、组合七社群共创
**不宜搭配**：企业官网、保守受众

---

#### S7 · 便当网格（Bento Grid）

**核心特征**：模块化瓷砖布局、不同大小卡片组合、有序但不死板的视觉层级、微交互增强
**最佳搭配**：多产品展示、功能介绍、组合五知识专家
**不宜搭配**：极简主义、长叙事型

---

#### S8 · 高端精致（Luxury Refined）

**核心特征**：Didot/Bodoni 高端衬线、黑白金极简配色、电影级影像、奢侈留白
**最佳搭配**：组合六高端精品
**不宜搭配**：大众消费品、信息密集型

---

#### S9 · 活力多彩（Vibrant Colorful）

**核心特征**：高饱和渐变、圆润形状、插画元素可选、粉彩或霓虹、友好感
**最佳搭配**：年轻品牌、组合七社群共创、儿童/教育
**不宜搭配**：高端精品、企业权威型

---

#### S10 · 中式雅韵（Chinese Elegant）

**核心特征**：水墨/书法元素现代化运用、传统纹样点缀、宋体标题增加文化厚度、留白即意境、中国红/墨色/山水色系
**最佳搭配**：文化/茶/中医/传统手工、组合八孝心型
**不宜搭配**：科技产品、年轻潮流品牌

### 风格选择规则

1. 每个项目选择 **1 个主风格 + 最多 1 个辅风格**
2. 主风格决定整体视觉语言（色彩、字体、布局密度）
3. 辅风格仅用于特定区块（如 Hero 用 S5 摄影沉浸，内容区用 S2 企业专业）
4. **冲突组合禁止同时选**：
   - S1 极简 × S9 活力多彩 / S21 极繁（留白 vs 填充）
   - S4 科技暗黑 × S3 温暖人文（冷峻 vs 温暖）
   - S8 高端精致 × S7 便当网格 / S15 粗野主义（极简 vs 密集/反叛）
   - S19 手绘 × S20 瑞士排版（有机 vs 理性）
   - S25 有机 × S26 几何（曲线 vs 直线）
5. 风格选择必须与**定位约束矩阵**兼容（见各组合"推荐风格搭配"）
6. S11-S27 中部分风格有可访问性/性能限制，使用前确认目标受众

### 色彩差异化规则（防止千篇一律）

每个新项目生成色彩方案时：

1. **先看行业色相惯例**，再决定沿用（专业感）还是反其道（差异化）
2. **同一行业连续两个项目，主色相必须不同**
3. **"温暖"≠ 米白+棕色**，温暖色系有 20+ 种选择：雾霾蓝、浅灰绿、淡金棕、月白、麦秆黄、玫瑰灰...
4. **对每个新项目，在设计文档中明确写出"为什么不用上一个案例的配色"**

⚠️ 禁止在任何新项目中直接使用以下组合（蒋苏站已用，禁止复用）：
主色 `#0F0F0F` + 辅色 `#FF6B35` + 背景 `#FAF7F2`

## 第三步：页面积木库

页面数量和组合**由设计文档决定**，不预设固定结构。以下是可选页面积木：

### 必选
| 页面 | 文件名 | 核心目标 |
|------|--------|---------|
| 首页 / 落地页 | `index.html` | 10秒内传达"你是谁、能帮我什么、我该怎么做" |

### 个人IP专属（A1/A3 必须有）
| 页面 | 文件名 | 核心目标 |
|------|--------|---------|
| 关于 / 故事页 | `about.html` | 打透"你是谁"，建立情感连接 |

### 按需选配
| 页面 | 文件名 | 适用条件 | 核心目标 |
|------|--------|---------|---------|
| 产品 / 服务页 | `services.html` | 有 2 条以上差异化产品线 | 帮用户选到合适方案 |
| 案例 / 作品页 | `cases.html` | 有实际可展示的成果 | 用成果消除购买顾虑 |
| 社群 / 加入页 | `community.html` | 有会员/订阅/付费社群 | 让犹豫者迈出加入那一步 |
| 内容 / 资源页 | `resources.html` | 有内容矩阵或免费资源 | 建立专业感，吸引潜在用户 |
| 预约 / 联系页 | `contact.html` | CTA 是表单或预约 | 降低行动门槛，收集意向 |
| 市场洞察页 | `insights.html` | 有行业权威内容要展示 | 建立专业权威 |
| 活动页 | `activities.html` | 有线下活动或定期活动 | 展示活动现场，吸引参与 |

### 典型组合参考
| 场景 | 页面组合 |
|------|---------|
| 轻量个人IP（刚起步） | 首页 + 关于 + 联系 |
| 完整个人IP | 首页 + 关于 + 产品/服务 + 社群 + 资源 |
| 企业品牌（B端主） | 首页 + 服务 + 案例 + 联系 |
| 企业品牌（B2C主） | 首页 + 产品 + 案例 + 市场洞察 |
| 创意空间/体验馆 | 首页 + 关于 + 活动 + 联系 |
| 知识付费IP | 首页 + 关于 + 产品/课程 + 资源 + 联系 |

## 第四步：Section 布局模式库

**核心规则：相邻 section 不得使用相同的布局模式。**

### 基础模式（12种）

| 模式 | 适用内容 | 节奏特征 |
|------|---------|---------|
| 全幅 Hero | 首屏开场 | 冲击感，必有 |
| 等宽卡片网格 | 功能亮点、并列服务（3-4 个等重要的项） | 规整标准，每页只用 1-2 次 |
| 交替左右行 | 痛点/价值观/服务步骤（有叙事关系） | 节奏感强，适合娓娓道来 |
| 锯齿形时间线 | 流程步骤、发展历程 | 动感，有方向感 |
| 杂志式不对称 | 数据展示、混合内容 | 不对称，高级感 |
| 照片分隔带 | section 之间插入，打破节奏 | 氛围感、呼吸感，每页至少 1 个 |
| 特色+次要组合 | 客户评价、案例展示 | 1 大 + 3 小，有主次 |
| 标签页切换 | 多类别产品/服务 | 信息密度高但整洁 |
| 全幅交替行 | 企业文化、价值观 | 沉浸感，水印编号 |
| 渐进式大小 | 版本演进、阶段对比 | 视觉权重递增 |
| 视差背景条 | CTA 区或 section 间的氛围强化 | 沉浸感，视觉重量 |
| FAQ 手风琴 | 常见问题 | 节省空间，按需展开 |

### 补充模式（2025-2026趋势）

| 模式 | 适用内容 | 节奏特征 |
|------|---------|---------|
| 数字冲击条 | 核心数据（年限、客户数、满意率等 4-6 个关键数字） | 横排大数字 + 计数器动画，信任锚点，首页必选 |
| 品牌Logo滚动墙 | 合作客户 / 合作品牌（≥6个Logo有效） | 灰度→悬停彩色，无限滚动，信任背书但视觉克制 |
| 双路径分流 | B端/C端入口、业主/用户入口 | 左右对称各有独立CTA，适合 B4 混合受众 |

### ❌ 糟糕节奏（避免这样做）

```
Hero → 等宽卡片 → 等宽卡片 → 等宽卡片 → CTA
（全是卡片网格，3屏后用户视觉疲劳，放弃浏览）

Hero → 交替左右行 × 6 → CTA
（单一节奏重复，缺乏高潮和变化）
```

**规律**：同一布局模式连续出现超过2次 = 节奏崩溃。每3个section必须有一次视觉"呼吸点"。

### 五 section 页面示例节奏
```
Hero（全幅背景图）
↓ 痛点/共鸣（交替左右行）
↓ [照片分隔带]
↓ 解决方案（锯齿形时间线）
↓ 定价/产品（等宽卡片网格）
↓ 客户评价（特色+次要组合）
↓ CTA（视差背景条）
```

## 第五步：转化与信任设计规则

### 转化硬规则

| 规则 | 实操要求 |
|------|---------|
| CTA 必须在首屏可见 | Hero 区块必须包含主 CTA 按钮，不得仅在页尾出现 |
| 每页至少 2 个 CTA | Hero CTA + 页尾 CTA 为最低标准，中间按需追加 |
| CTA 文案用第一人称行动语言 | "我要咨询""立即加入"优于"了解更多""联系我们" |
| 信任证据占关键页面 30-50% | 评价、案例、数据、Logo墙、认证徽章累计面积 |
| 50ms 第一印象法则 | 首屏：清晰标题 + 高质量背景图 + 明确 CTA，不可模糊 |
| 表单字段尽量少 | 首次接触最多 3-5 个字段（姓名、联系方式、简述） |

### 信任元素优先级

| 优先级 | 信任元素 | 实施要点 |
|--------|---------|---------|
| P0 必选 | 带照片和量化成果的真实评价 | "合作3年，年营收增长40%" > "服务很好" |
| P0 必选 | 核心数字大展示 | 年限、客户数、满意率、项目数，计数器动画 |
| P1 强烈推荐 | 客户 Logo 墙 | 灰度→悬停彩色，≥6个品牌 |
| P1 强烈推荐 | 具名案例故事 | 包含客户姓名/公司+具体过程+量化结果 |
| P2 推荐 | 团队/主理人真实照片 | 真实场景照 > 证件照 > stock photo |
| P2 推荐 | 资质认证/媒体报道 | 官方背书、权威引用、行业认证 |
| P3 加分 | 反面案例/失败教训 | 罕见但极高可信度 |
| P3 加分 | 透明定价 | 价格清晰比"联系咨询"转化率更高（B2C场景） |

### 中国市场额外信任锚点

| 信任信号 | 说明 | 适用场景 |
|---------|------|---------|
| 企业微信官方认证 | 比普通二维码信任度高 | 所有B端/私域场景 |
| 国企/政府客户Logo | 国企背书在中国B端场景极高权重 | To B 服务商 |
| 媒体截图（公众号/小红书/抖音） | 公众号10W+、平台粉丝量截图 | 个人IP/内容创作者 |
| 真实微信对话截图 | 客户在微信里的真实反馈（打码处理） | 个人服务/咨询 |
| 价格透明+包含清单 | 中国消费者对隐藏收费极度反感 | B2C/知识付费 |

### 信息密度控制

| 模式 | 每屏信息点 | 留白比例 | 适用 |
|------|-----------|---------|------|
| **呼吸型** | 1-2 个重点 | 60%+ 留白 | B3 高净值、疗愈/设计/奢侈品 |
| **标准型** | 2-3 个重点 | 40-50% 留白 | 多数 B2C / 个人IP |
| **信息型** | 3-5 个重点 | 25-35% 留白 | B1 企业端、多产品线 |

## 视觉质量检查清单（七支柱）

### ⛔ 一票否决项（任一未达标 = 整页重做）

- [ ] 首页 Hero CTA 在首屏 fold 以上可见
- [ ] 主标题不是描述性的（"我们是XX公司"必须改为叙事性标题）
- [ ] 所有颜色通过 CSS 变量，无任何硬编码颜色字面量
- [ ] 导航栏在所有页面结构完全一致
- [ ] 首屏有真实视觉内容（禁止纯色背景作 Hero）

### 支柱一：视觉层级（50ms 法则）
- [ ] 首屏 3 秒内传达"你是谁 + 能帮什么 + 我该做什么"
- [ ] 首屏有全幅背景图或高质量视觉元素
- [ ] 首屏不超过 3 个对比层级（标题、副标题、正文）
- [ ] 主 CTA 是首屏视觉最突出的单一元素

### 支柱二：内容策略
- [ ] 每个页面有单一清晰目标
- [ ] 标题是叙事性的（"帮你省50万"）而非描述性的（"我们的服务"）
- [ ] 文案达到可直接上线的质量，非 Lorem Ipsum

### 支柱三：信任可视化
- [ ] 关键页面信任证据占比 ≥ 30%
- [ ] 评价使用真实风格（照片 + 姓名 + 具体成果）
- [ ] 数据有量化和出处（"6年服务200+企业"而非"经验丰富"）
